# Дашборд рассадки сотрудников

Локальное offline-приложение для планирования переезда сотрудников из старых офисов в новые: сценарии рассадки, офисы и зоны, команды и сотрудники, размещения с drag-and-drop, расчёты заполняемости и переполнения, KPI, валидация, сравнение сценариев и экспорт.

Технологии: чистые **HTML + CSS + Vanilla JavaScript**. Без backend, без CDN, без сборки. Работает offline.

---

## Запуск

### Вариант A — готовый файл для распространения (рекомендуется)

Скачайте файл **`employee-seating-dashboard.html`** из корня репозитория и откройте двойным кликом.  
Всё вшито внутрь одного файла (~2.9 МБ): CSS, JavaScript, библиотеки Excel/PDF/PNG.  
Работает offline. Не нужны Node.js, веб-сервер, интернет.

> Именно этот файл удобнее всего передавать коллегам.

### Вариант B — из исходников

1. Скачайте всю папку репозитория (содержит `index.html`, `js/`, `libs/`, `styles.css`).
2. Откройте файл **`index.html`** двойным кликом.

> Если браузер блокирует загрузку скриптов из `file://` — используйте Вариант A.

### Пересборка бандла после изменений исходников

```
python build.py
```

Генерирует `employee-seating-dashboard.html` заново из актуальных исходников.

### Локальные библиотеки (папка `libs/`)

Для Excel/PDF/PNG используются локальные библиотеки в `libs/`:

| Файл | Назначение |
|------|-----------|
| `libs/xlsx.full.min.js` | импорт/экспорт Excel (SheetJS) |
| `libs/jspdf.umd.min.js` | экспорт PDF (jsPDF) |
| `libs/pdf-fonts.js` | шрифт PT Sans с кириллицей для PDF (SIL OFL 1.1, см. `libs/PTSans-OFL.txt`) |
| `libs/html2canvas.min.js` | экспорт PNG (html2canvas) |

Эти файлы уже включены в комплект. Приложение запрашивает их только локально из `libs/` — обращений в интернет нет.

> PDF формируется с кириллическим шрифтом PT Sans (встроен в `pdf-fonts.js`), поэтому русский текст отображается корректно. Если `pdf-fonts.js` удалить, PDF продолжит работать со встроенным шрифтом jsPDF (только латиница).
Если какой-то файл отсутствует, приложение продолжит работать, а при попытке соответствующего экспорта покажет подсказку, какой файл и куда положить. Остальной функционал (сценарии, drag-and-drop, расчёты, JSON) от этого не зависит.

## Тесты

Откройте **`tests.html`** двойным кликом в браузере — автоматически прогонятся юнит-тесты расчётов и валидации. Зелёная плашка «Пройдено N из N» означает, что формулы вместимости/переполнения/KPI и коды сообщений работают корректно.

## Импорт списка сотрудников из Excel

Чтобы не добавлять сотрудников вручную, загрузите их из Excel-файла (`.xlsx`).

### Где это в интерфейсе

Вкладка **«Отчеты»**, блок **«Excel»**:

- **«Скачать шаблон Excel»** — пустой файл `seating-template.xlsx` с нужными листами и заголовками.
- **«Импорт Excel»** — загрузить заполненный файл.

В комплекте также есть готовый пример с данными — **`sample-data.xlsx`** (в корне папки приложения): откройте его, чтобы увидеть, как заполнять, или сразу импортируйте для проверки.

### Порядок действий

1. Нажмите **«Скачать шаблон Excel»** (или возьмите `sample-data.xlsx` как образец).
2. Заполните лист **`Employees`** (при необходимости — `Teams`, `Offices`, `Zones`).
3. Вкладка «Отчеты» → **«Импорт Excel»** → выберите файл.
4. В появившемся окне выберите **«Новый сценарий»** или **«Обновить текущий»**.
5. После импорта откроется отчёт: сколько импортировано офисов/зон/команд/сотрудников и сколько ошибок.

### Лист `Employees` — колонки

Поддерживаются и русские, и английские заголовки (можно использовать любые):

| Колонка (EN) | Колонка (RU) | Обяз. | Пример |
|---|---|:---:|---|
| `full_name` | `ФИО` | **да** | Иванов Иван Иванович |
| `position` | `Должность` | нет | Аналитик |
| `team_name` | `Команда` | нет | Finance |
| `current_office` | `Текущий офис` | нет | Старый офис A |
| `cabinet` | `Кабинет` | нет | Кабинет 301 |
| `is_vip` | `VIP` | нет | да / нет |
| `work_format` | `Формат работы` | нет | офис / гибрид / удаленка |
| `comment` | `Комментарий` | нет | — |

Шаблон содержит ещё три листа для справочных данных:

- **`Teams`**: `team_name`/`Название команды`, `employees_count`/`Количество сотрудников`, `current_office`/`Текущий офис`, `is_vip`/`VIP`, `can_split`/`Можно делить`, `comment`/`Комментарий`.
- **`Offices`**: `office_name`/`Название офиса`, `office_type`/`Тип офиса` (`old`/`старый` или `new`/`новый`), `area`/`Площадь`, `capacity`/`Вместимость` (для старого офиса), `cabinet_capacity`/`Кабинеты`, `open_space_capacity`/`Опенспейс`, `vip_capacity`/`VIP-кабинеты` (для нового офиса — превращаются в зоны), `is_draft`/`Черновик`, `comment`/`Комментарий`.
- **`Zones`**: `office_name`/`Название офиса`, `zone_name`/`Название зоны`, `zone_type`/`Тип зоны`, `capacity`/`Вместимость`, `is_vip_zone`/`VIP-зона`, `comment`/`Комментарий` — для офисов с несколькими зонами.

### Важные нюансы

- Обязательно только поле **`ФИО`**. Строки с пустым ФИО не импортируются и попадают в отчёт об ошибках; остальные загружаются.
- **`Команда`** сопоставляется по названию с командой из листа `Teams` (или уже существующей в сценарии). Если команда не найдена — сотрудник загрузится без команды.
- **`Текущий офис`** сопоставляется по названию с офисом из листа `Offices`/сценария.
- **`Формат работы`**: `офис`/`office`, `гибрид`/`hybrid`, `удаленка`/`remote`.
- **`VIP`**: `да`/`yes`/`1`/`x` — VIP, иначе обычный.
- **Дубликаты** (одинаковые названия офисов/команд) считаются ошибкой и пропускаются.
- **Размещения**: если у сотрудника заполнен `Текущий офис` или `Кабинет`, при импорте автоматически создаётся индивидуальное размещение в соответствующий офис/зону. Остальные размещения расставляются в приложении через drag-and-drop.
- Импорт фиксируется как одно действие — его можно отменить (Ctrl+Z).

## Возможности

- **Сценарии** (слева): создание, дублирование, переименование, комментарий, удаление (с подтверждением; последний удалить нельзя), переключение, сравнение.
- **Офисы**: старый офис, новый офис (с зонами), системный офис «Удаленка». У нового офиса всегда есть хотя бы одна зона (если не создана — автоматически «Опенспейс»). Стандартные зоны: Кабинеты, Опенспейс, VIP-кабинеты; плюс кастомные. Признак черновика.
- **Команды**: численность, текущий офис, VIP, «можно делить», остаток, «отправить остаток на удаленку».
- **Сотрудники**: ФИО, должность, команда, текущий офис, VIP, формат работы (офис/гибрид/удаленка); поиск по ФИО и фильтры (команда, текущий/новый офис, VIP, формат, статус размещения). Приложение работает и без списка сотрудников — только по командам и численности.
- **Размещения**: командные и индивидуальные; ручная таблица размещений; редактирование и удаление; сотрудник всегда занимает 1 место.
- **Drag-and-drop** (вкладки «Дашборд» и «Распределение»): перетаскивание команд (с popup выбора количества и подсказкой остатка), частей команд и отдельных сотрудников; предупреждения при VIP-конфликтах; клавиатурная альтернатива (Space/Enter — взять/положить, Esc — отмена) и анонсы для скринридеров.
- **Расчёты**: вместимость офиса = сумма зон; занятость по размещениям; переполнение в формате `Переполнение: N мест`; заполненность в процентах; переполнение офисов и зон считаются **раздельно**.
- **KPI и дашборд**: KPI-блок, прогресс переезда, карточки офисов (компактные/расширенные) с progress bars, карточка «Удаленка», список команд, «Проблемные офисы», панель предупреждений.
- **Цвета и пороги**: зелёный ≤ 85%, жёлтый 85–100%, красный > 100%, серый — нет данных, синий — информация. Оба порога редактируются в «Настройках» и сохраняются в проекте.
- **Импорт/экспорт**: JSON (основной формат проекта), Excel (шаблон/импорт/экспорт, листы `Offices/Zones/Teams/Employees` при импорте и `Summary/Offices/Zones/Teams/Employees/Allocations/Warnings` при экспорте; RU/EN заголовки колонок), PDF (сравнение, KPI, карточки, предупреждения; переключатель «включать ФИО»), PNG (весь дашборд), печать браузера.
- **Режим «Только просмотр»**: блокирует добавление/редактирование/удаление/drag-and-drop/импорт; экспорт и просмотр доступны.
- **Undo/Redo**: Ctrl+Z / Ctrl+Y (или Ctrl+Shift+Z) для добавления, удаления, редактирования, drag-and-drop. Импорт фиксируется как одно крупное действие.
- **Хранение**: опциональное автосохранение в `localStorage` (включается в «Настройках») + JSON-экспорт/импорт как основной способ.
- **Onboarding** при первом запуске (кнопка «?» вызывает его снова) и понятные пустые состояния.

### Сущности

- **office.type**: `old` (старый офис — справочник, без зон, не участвует в расчёте новых офисов), `new` (новый офис с зонами), `remote` (системная «Удаленка», без лимита).
- **zone.type**: `cabinet` | `open_space` | `vip`. `isVipZone` — признак VIP-зоны. `isSystem` — служебная зона (например авто-«Опенспейс»). Для VIP: 1 кабинет = 1 место (вводится числом мест).
- **team**: `employeesCount` — численность; `canSplit` — можно ли делить; `isVip` — VIP/руководство; `currentOfficeId` — текущий офис.
- **employee.workFormat**: `office` | `hybrid` | `remote`. Гибрид учитывается справочно и не влияет на расчёт мест. FTE не используется — все сотрудники целые единицы.
- **allocation.type**: `team` (часть/вся команда, `employeesCount` ≥ 1) или `employee` (`employeesCount` всегда 1). `targetOfficeId`/`targetZoneId` — куда направлено (для удаленки `targetZoneId` = `null`).

### Ключевые формулы

- Вместимость офиса = сумма вместимости его зон (удаленка — без лимита; старый офис не считается).
- Занятость офиса/зоны = сумма `employeesCount` размещений в офис/зону.
- Переполнение = `занято − вместимость` (показывается только при > 0, формат `Переполнение: N мест`).
- Заполненность (%) = `занято / вместимость × 100`.
- Требуется посадочных мест = `всего сотрудников − на удаленке`.
- Свободный резерв = `вместимость новых офисов − размещено в офисных зонах`.
- Переполнение офисов и переполнение зон считаются и показываются **раздельно**.
