# Prompt для ИИ-агента

Ты — senior frontend developer. Создай локальное offline-веб-приложение **«Дашборд рассадки сотрудников»** для планирования переезда сотрудников из старых офисов в новые.

## Главная цель

Создать интерактивный dashboard-калькулятор, который позволяет:

- создавать сценарии переезда;
- добавлять офисы, зоны, команды, сотрудников;
- распределять команды, части команд и сотрудников по офисам и зонам;
- использовать drag-and-drop;
- видеть заполняемость, переполнение, ошибки и предупреждения;
- сравнивать сценарии;
- импортировать и экспортировать данные.

## Технологии

- HTML;
- CSS;
- Vanilla JavaScript;
- без backend;
- без внешних API;
- без CDN;
- приложение должно работать offline;
- приложение должно запускаться двойным кликом по `index.html`;
- не использовать ES-модули, если они мешают запуску через `file://`;
- внешние библиотеки подключать только локально из папки `libs/`.

- Распределение;
- Офисы;
- Команды;
- Сотрудники;
- Сравнение сценариев;
- Отчеты.

Слева должна быть боковая панель сценариев.

## Сценарии

Каждый сценарий — самостоятельная модель:

- офисы;
- зоны;
- команды;
- сотрудники;
- размещения;
- комментарий.

Нужны функции создания, дублирования, переименования, удаления и сравнения сценариев.

Последний сценарий удалить нельзя.

## Офисы

Типы офисов:

1. Старый офис.
2. Новый офис.
3. Системный офис `Удаленка`.

Старый офис: название, площадь, текущая вместимость, комментарий. Зон нет.

Новый офис: название, площадь, зоны и вместимость, комментарий, признак черновика.

Каждый новый офис должен иметь хотя бы одну зону. Если пользователь не создал зоны, автоматически создать зону `Опенспейс`.

Стандартные зоны:

- Кабинеты;
- Опенспейс;
- VIP-кабинеты.

Можно добавлять кастомные зоны.

Удаленка — системный офис без лимита вместимости. Удаленка не считается физическим офисом и не участвует в расчете вместимости новых офисов.

## Команды

Поля команды:

- название;
- количество сотрудников;
- текущий офис;
- VIP / руководство;
- можно делить;
- комментарий.

Команду можно делить, если включена настройка `Можно делить`.

VIP-зона доступна всем, но конфликтные случаи показываются как предупреждения.

## Сотрудники

Поля сотрудника:

- ФИО;
- должность;
- команда;
- текущий офис;
- VIP / руководство;
- формат работы: офис / гибрид / удаленка;
- комментарий.

Гибридный формат не влияет на расчеты.

FTE не использовать.

Приложение должно работать и без списка сотрудников — только по командам и численности.

## Размещения

Размещение — запись о том, куда направлена команда, часть команды или конкретный сотрудник.

Поля:

- тип: команда / сотрудник;
- ID команды или сотрудника;
- количество;
- офис назначения;
- зона назначения;
- комментарий.

Для сотрудника количество всегда равно `1`.

## Drag-and-drop

Drag-and-drop обязателен и не должен быть упрощен.

Перетаскивать можно:

- команду;
- часть команды / allocation-блок;
- сотрудника.

При перетаскивании команды открывать popup с выбором количества. Если команда уже частично размещена, предлагать остаток.

При перетаскивании сотрудника назначать его сразу, но показывать предупреждения при VIP-конфликтах.

## Ручное редактирование

Сделать таблицу размещений для точного ручного редактирования.

## Расчеты

Вместимость офиса = сумма вместимости зон.

Занято в офисе = сумма сотрудников в зонах офиса.

Свободно = вместимость - занято.

Переполнение = занято - вместимость, если значение больше 0.

Переполнение обязательно показывать числом:

```text
Переполнение: 17 мест
```

Заполненность (%) = занято / вместимость × 100.

Для зоны считать аналогично.

KPI на главном дашборде:

- Всего сотрудников;
- Требуется посадочных мест;
- Вместимость новых офисов;
- Распределено в офисы;
- На удаленке;
- Не размещено;
- Свободный резерв;
- Переполнение офисов;
- Переполнение зон;
- Количество предупреждений;
- Количество ошибок.

Требуется посадочных мест = все сотрудники - сотрудники в удаленке.

Показывать переполнение офисов и переполнение зон отдельно.

## Цвета

Пороги редактируемые.

Дефолт:

- зеленый до 85%;
- желтый 85–100%;
- красный выше 100%;
- серый нет данных;
- синий информационный статус.

## Валидация

Ошибки:

- команда распределена больше численности;
- сотрудник размещен дважды.

Предупреждения:

- офис переполнен;
- зона переполнена;
- сотрудник не размещен;
- команда не полностью распределена;
- обычная команда / сотрудник размещены в VIP-зоне;
- VIP-команда / VIP-сотрудник размещены не в VIP-зоне;
- черновой офис с нулевой вместимостью используется.

Информация:

- сотрудник на удаленке;
- есть сотрудники на удаленке.

## Дашборд

Главный экран должен содержать:

- KPI-блок;
- общий прогресс переезда;
- карточки новых офисов;
- карточку `Удаленка`;
- список команд;
- панель предупреждений;
- блок `Проблемные офисы`.

Карточка офиса имеет компактный и расширенный режим.

Компактная карточка:

- название;
- площадь;
- вместимость;
- занято;
- свободно / `Переполнение: N мест`;
- заполненность %;
- progress bar;
- предупреждения.

Расширенная карточка:

- зоны с progress bar;
- состав по командам;
- раскрытие сотрудников;
- редактирование.

## Сотрудники

Во вкладке `Сотрудники` сделать таблицу, поиск и фильтры:

- по ФИО;
- по команде;
- по текущему офису;
- по новому офису;
- по VIP;
- по формату работы;
- по статусу размещения.

## Импорт Excel

Поддержать `.xlsx`.

Шаблон содержит листы:

- Offices;
- Zones;
- Teams;
- Employees.

Размещения из Excel не импортировать.

Поддерживать русские и английские названия колонок.

При импорте пользователь выбирает: создать новый сценарий или обновить текущий.

Корректные строки импортировать, ошибочные показывать отдельно.

После импорта показать отчет.

## Экспорт

Реализовать:

- JSON export/import;
- Excel export;
- PDF export;
- PNG export;
- печать через браузер.

JSON должен сохранять весь проект: сценарии, офисы, зоны, команды, сотрудники, размещения, настройки и пороги.

Excel содержит листы:

- Summary;
- Offices;
- Zones;
- Teams;
- Employees;
- Allocations;
- Warnings.

PDF содержит сравнение сценариев, KPI, карточки офисов, предупреждения и ошибки.

Добавить переключатель `Включать персональные данные сотрудников`.

PNG экспортирует весь дашборд активного сценария.

## Режим только просмотра

Обязателен.

В этом режиме запрещены редактирование, удаление, добавление, drag-and-drop и импорт.

Экспорт и просмотр доступны.

## Undo / Redo

Обязательно.

Отменять все изменения, кроме импорта.

## Onboarding и пустые состояния

При первом запуске показать краткий onboarding.

Если данных нет, показывать понятные empty states.
